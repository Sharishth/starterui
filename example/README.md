# StarterUI Example Page

This is a showcase example demonstrating various StarterUI components in a real-world scenario.

## What's Included

This example page demonstrates the following StarterUI components:

### Navigation
- **Responsive Navbar** - Sticky navigation with mobile menu support
- **Breadcrumb** - Navigation breadcrumb in hero section

### Layout Components
- **Hero Section** - Full-screen hero with gradient background
- **Grid Layout** - Stats section using grid
- **Flex Layout** - Multiple flex-based layouts throughout

### UI Components
- **Buttons** - Various button styles and colors
- **Cards** - Text cards, quote cards, and pricing cards
- **Badges** - Count, success, info, and warning badges
- **Accordion** - FAQ section with expandable content
- **Carousel** - Portfolio showcase carousel
- **Tabs** - Services section with tab navigation
- **Timeline** - Journey/progress timeline
- **Form Elements**:
  - Text inputs
  - Email inputs
  - Dropdown/Select
  - Range slider
  - Toggle switch
  - Radio buttons

### Other Components
- **Code Block** - Syntax highlighted code example
- **Footer** - Grid-based footer layout
- **Dividers** - Visual separators
- **Shadows** - Box shadows for depth

## Features Demonstrated

1. **Responsive Design** - Works on mobile, tablet, and desktop
2. **Interactive Elements** - Carousel, tabs, accordion, dropdowns
3. **Modern Design** - Gradient backgrounds, shadows, and animations
4. **Form Handling** - Complete contact form with various input types
5. **Component Combinations** - Shows how components work together

## How to View

Simply open `index.html` in a web browser. The page uses CDN links for StarterUI, so an internet connection is required.

## Customization

Feel free to modify this example to:
- Change colors and gradients
- Add more sections
- Customize component styles
- Add your own content
- Integrate with a backend

## Notes

- All JavaScript functionality is provided by StarterUI's min.js
- Components are styled using StarterUI's utility classes
- Additional custom CSS is included in the `<style>` tag for page-specific styling
- The page is fully functional and interactive
